# Softr App Specification
**BOS | Business Opportunity Systems — Funding Intelligence Vault**
**Web App Version — Built on Softr + Airtable**

---

## Overview

The Softr web app is the premium delivery format for the Funding Intelligence Vault. Instead of a spreadsheet file, buyers receive a URL to a fully functional web application — searchable, filterable, and accessible from any device. No downloads, no "make a copy," no setup.

**Stack:**
- **Data:** Airtable base (per `airtable-base-spec.md`)
- **App layer:** Softr (softr.io)
- **Access control:** Softr magic-link login (buyer's email added after purchase)
- **Branding:** Custom domain optional; Softr subdomain fine for v1

---

## 1. Airtable Setup (Prerequisite)

Before building in Softr, complete the Airtable base setup per `airtable-base-spec.md`.

**Key requirements for Softr integration:**
- Base must use **Airtable API** (not Airtable Interfaces — Softr has its own layer)
- All select fields must use single/multiple select type (not text)
- The **Status** field must be a single-select with color coding
- **Fit Score** and **Difficulty** must be single-select fields
- **Website** must be a URL field (not plain text)

## 2. Softr Account & App Setup

### Create the app

1. Go to [softr.io](https://softr.io) and sign in / create account
2. Click **Create new app**
3. Choose **Start from scratch** (not a template — we'll build custom)
4. App name: `Funding Intelligence Vault`
5. Connect data source: **Airtable**
6. Select your Airtable base and the `Funding Opportunities` table

### Branding

| Setting | Value |
|---|---|
| App name | Funding Intelligence Vault |
| Brand color | `#F59E0B` (amber) |
| Secondary color | `#0F172A` (navy) |
| Font | Inter |
| Logo | Text: "Funding Intelligence Vault" (or upload SVG logo) |
| Favicon | Upload a small vault/lock icon |

## 6. Access Control (Login / Gating)

### Setup

1. In Softr: **Settings → Access control**
2. Enable **User authentication**
3. Set login method: **Magic link** (email — no password required)
4. All pages: Set visibility to **Logged-in users only** (except a public landing page if desired)

### Adding buyers

After each purchase:
1. Go to Softr **Users** panel
2. Click **Add user**
3. Enter the buyer's email address
4. Optional: Send them a welcome email from Softr

**Automation option (recommended):**
Connect Formspree (or your email capture) → Zapier/Make → Softr API to auto-add users after purchase.

### Softr Users API

```
POST https://api.softr.io/v1/applications/{app_id}/users
Authorization: Bearer {softr_api_key}
Content-Type: application/json

{
  "email": "buyer@example.com",
  "full_name": "Buyer Name",
  "activated": true
}
```

## 7. Custom Domain (Optional)

1. Purchase a domain or use a subdomain: e.g. `vault.itsdigitally.com`
2. In Softr: **Settings → Custom domain**
3. Add CNAME record: `vault` → `[your-softr-app].softr.app`
4. Wait for SSL propagation (15–60 minutes)
5. Update `config.js` → `softrAppUrl: "https://vault.itsdigitally.com"`

## 10. Buyer Experience (What They See)

1. Complete purchase on your site
2. Receive welcome email: "Your Funding Intelligence Vault is ready — click here to access"
3. Email contains magic link → clicks → instantly logged in, no password
4. Lands on **Dashboard** — sees welcome message, stats, and top 6 picks
5. Clicks **Funding Directory** — searches, filters, browses
